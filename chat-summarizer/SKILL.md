# Chat Summarizer Skill

Produces a well-structured `.md` file summarizing the current chat session.

## Workflow

### Step 1 — Extract conversation content
Read the full conversation from the current context window. Identify:
- The main topic(s) discussed
- Key decisions, conclusions, or outcomes
- Any code, commands, or files produced
- Action items or next steps mentioned by the user

### Step 2 — Build the summary document

Use this structure:

```markdown
# Chat Summary: <short topic title>

**Date:** <today's date>  
**Session Length:** <approximate, e.g. "~15 messages">

---

## Overview
<2–4 sentence high-level description of what the session covered and why>

## Topics Discussed
<bullet list of major subjects, in order they appeared>

## Key Outcomes
<What was produced, decided, or resolved. Be concrete — name files, algorithms, approaches.>

## Code / Files Produced
<If any code blocks, scripts, or files were created, list them here with a one-line description each. Omit if none.>

## Action Items / Next Steps
<Any tasks the user mentioned wanting to do next. Omit section if none.>

---
*Summary generated by Claude*
```

### Step 3 — Write the file

- **Filename:** derive a short slug (2–5 words, kebab-case) from the session's dominant topic, then append the date:
  `<topic-slug>-<YYYY-MM-DD>.md`
  Examples: `ppo-frozenlake-report-2026-04-10.md`, `battleship-sdl2-architecture-2026-04-10.md`, `chat-summary-2026-04-10.md` (fallback if topic is unclear)
- Save to `/mnt/user-data/outputs/`
- After saving, run `scripts/save_to_env_dir.py <output_path>` to also copy the file to the directory specified in `.env`
- Then call `present_files` so the user can download it

## Style Notes
- Keep the Overview concise — it's a recap, not a transcript
- Use bullet points for Topics and Action Items
- In Key Outcomes, prefer specifics over vague summaries ("Generated a PPO training loop in PyTorch" not "Discussed RL")
- Do not reproduce long code blocks in the summary; just reference them by name/purpose
